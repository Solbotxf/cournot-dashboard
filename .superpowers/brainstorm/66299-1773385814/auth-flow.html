Authentication Flow

Wallet connect → signature → JWT token lifecycle

Login Sequence
 Browser (wagmi)              Next.js Proxy             Cournot Backend
 ──────────────              ─────────────             ───────────────
       │                           │                          │
  1. User clicks                   │                          │
     "Connect Wallet"              │                          │
       │                           │                          │
  2. wagmi useConnect()            │                          │
     → MetaMask popup              │                          │
     → User approves               │                          │
     → Get address                 │                          │
       │                           │                          │
  3. POST /api/proxy/auth/challenge                           │
     { address: "0xABC..." }  ────►│                          │
       │                           │  POST /auth/challenge    │
       │                           │  { address: "0xABC..." } │
       │                           │ ────────────────────────► │
       │                           │                          │
       │                           │  { challenge:            │
       │                           │    "Sign this message    │
       │                           │     to login to Cournot  │
       │                           │     Dashboard\n          │
       │                           │     Nonce: a1b2c3\n      │
       │                           │     Timestamp: 2026-...  │
       │                           │     Expires: 300s"       │
       │                      ◄────│  }                       │
       │  ◄────────────────────────│                          │
       │                           │                          │
  4. wagmi useSignMessage()        │                          │
     → MetaMask sign popup         │                          │
     → User signs                  │                          │
     → Get signature               │                          │
       │                           │                          │
  5. POST /api/proxy/auth/verify                              │
     { address: "0xABC...",   ────►│                          │
       signature: "0x123...",      │  POST /auth/verify       │
       challenge: "Sign..." }      │  { address, signature,   │
       │                           │    challenge }           │
       │                           │ ────────────────────────► │
       │                           │                          │
       │                           │  Backend:                │
       │                           │  - ecrecover(sig) →addr  │
       │                           │  - verify nonce+timestamp│
       │                           │  - lookup role for addr  │
       │                           │  - generate JWT          │
       │                           │                          │
       │                           │  { token: "eyJ...",      │
       │                           │    address: "0xABC...",  │
       │                           │    role: "admin",        │
       │                      ◄────│    expires_at: "..." }   │
       │  ◄────────────────────────│                          │
       │                           │                          │
  6. Store in AuthContext           │                          │
     + localStorage:               │                          │
     - token (JWT)                 │                          │
     - address                     │                          │
     - role                        │                          │
       │                           │                          │
  7. All subsequent requests       │                          │
     include header:               │                          │
     Authorization: Bearer eyJ...  │                          │
       │                           │                          │

Auth API Spec

POST /auth/challenge

Request a challenge message for wallet signature. Contains nonce + timestamp to prevent replay.

Request 
{ "address": "0xABC..." }
Response 
{
  "challenge": "Sign this message...",
  "expires_at": "2026-03-13T..."
}
POST /auth/verify

Submit signed challenge. Backend recovers address from signature, verifies nonce, returns JWT + role.

Request 
{
  "address": "0xABC...",
  "signature": "0x123...",
  "challenge": "Sign this..."
}
Response 
{
  "token": "eyJhbG...",
  "address": "0xABC...",
  "role": "admin" | "user",
  "expires_at": "2026-03-14T..."
}
GET /auth/me

Validate existing JWT and return current session. Used on page reload to restore auth state.

Header 
Authorization: Bearer eyJhbG...
Response 
{
  "address": "0xABC...",
  "role": "admin" | "user",
  "expires_at": "2026-03-14T..."
}
POST /auth/logout

Invalidate JWT server-side. Frontend also clears localStorage and disconnects wallet.

Header 
Authorization: Bearer eyJhbG...
Response 
{ "ok": true }

Test Mode (NEXT_PUBLIC_ENABLE_TEST_AUTH=true)

When enabled, the "Connect Wallet" button bypasses MetaMask entirely:

  • Auto-sets address to 0xTEST...1234
  • Skips challenge/signature flow
  • Calls /auth/verify with a special test signature
  • Backend recognizes test mode and returns admin JWT
  • All subsequent API calls work normally with the test JWT

Frontend Auth State

AuthContext provides:
├── isConnected: boolean      ← wagmi wallet connected
├── isAuthenticated: boolean  ← JWT token valid
├── address: string | null    ← wallet address
├── role: "admin" | "user" | null
├── token: string | null      ← JWT for API calls
├── login()                   ← trigger challenge → sign → verify
├── logout()                  ← clear token + disconnect wallet
└── isLoading: boolean        ← auth flow in progress

localStorage keys:
├── "cournot_auth_token"      ← JWT string
├── "cournot_auth_address"    ← address for display
└── "cournot_auth_role"       ← role for quick UI checks

On page load:
1. Check localStorage for token
2. If found → GET /auth/me to validate
3. If valid → restore AuthContext
4. If invalid → clear localStorage, show disconnected
