Admin Market Management вҖ” API Spec

CRUD endpoints for admin-managed markets + resolution workflow

Market Data Model

AdminMarket {
  id:              string       // server-generated UUID
  title:           string       // "Will BTC hit $100k by June 2026?"
  platform:        string       // "polymarket" | "kalshi" | "limitless" | "custom"
  platform_url:    string?      // link to the market on the platform
  description:     string       // detailed market description
  question:        string       // the resolvable question (fed to PoR)

  start_time:      ISO 8601     // when the market opens
  end_time:        ISO 8601     // when the market closes for trading
  resolve_time:    ISO 8601     // when the market should be resolved

  status:          enum         // see status lifecycle below
  alert_level:     enum         // "none" | "low" | "medium" | "high" | "critical"
  alert_reason:    string?      // why the alert was triggered

  resolution:      object?      // null until resolved, see below
  por_result:      object?      // raw PoR RunSummary if triggered

  created_by:      string       // admin address who added it
  created_at:      ISO 8601
  updated_at:      ISO 8601
}

Resolution Object (attached when status = RESOLVED):
  resolution: {
    outcome:         string     // "YES" | "NO" | "INVALID" | custom
    confidence:      number     // 0-1, from PoR or admin override
    resolved_by:     string     // admin address
    resolved_at:     ISO 8601
    method:          string     // "por" | "manual" | "por_modified"
    por_root:        string?    // cryptographic PoR root if triggered
    admin_notes:     string?    // admin's reasoning for override
    evidence_summary: string?   // brief evidence summary
  }

Admin Endpoints

All admin endpoints require Authorization: Bearer {jwt} with role = "admin"

GET /admin/markets

List all admin-managed markets with filtering and pagination.

Query Params 
?status=ALERTED,MONITORING
&alert_level=high,critical
&platform=polymarket
&page=1
&page_size=20
&sort=resolve_time
&order=asc
Response 
{
  "markets": [AdminMarket, ...],
  "total": 42,
  "page": 1,
  "page_size": 20
}
POST /admin/markets

Add a new market to be monitored. Backend starts scanning after start_time.

Request Body 
{
  "title": "BTC $100k by June",
  "platform": "polymarket",
  "platform_url": "https://...",
  "description": "Whether...",
  "question": "Will Bitcoin...",
  "start_time": "2026-03-13T...",
  "end_time": "2026-06-30T...",
  "resolve_time": "2026-07-01T..."
}
Response (201) 
{
  "market": AdminMarket
}
GET /admin/markets/:id

Get full market detail including alert info and PoR result if available.

Response 
{ "market": AdminMarket }
PUT /admin/markets/:id

Update market metadata. Cannot update once status is RESOLVED.

Request Body 
{ partial AdminMarket fields }
POST /admin/markets/:id/resolve вҳ… KEY ENDPOINT

Submit final resolution. Triggers Lark notification + Telegram broadcast. This is the admin's confirmed result (may differ from PoR).

Request Body 
{
  "outcome": "YES",
  "confidence": 0.95,
  "method": "por_modified",
  "por_root": "abc123...",
  "admin_notes": "PoR said NO but
    official source confirms YES",
  "evidence_summary": "Reuters
    reported on 2026-03-13..."
}
Response 
{
  "market": AdminMarket,
  "notifications": {
    "lark": true,
    "telegram": true
  }
}
Backend side-effects:
  • Sets market status вҶ’ RESOLVED
  • Sends Lark message: "Market [title] resolved as [outcome] by [address]"
  • Sends Telegram message to designated group with full evidence + proofs

Error Responses

Standard error envelope (matches existing pattern):
{
  "code": 4xxx,
  "msg": "Human-readable error",
  "data": null
}

Error codes:
  4000  Bad request (validation error, missing fields)
  4010  Unauthorized (no token or expired)
  4030  Forbidden (valid token but not admin role)
  4040  Not found (market ID doesn't exist)
  4090  Conflict (market already resolved)
  5000  Internal server error
