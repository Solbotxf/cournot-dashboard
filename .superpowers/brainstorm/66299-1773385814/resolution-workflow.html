PoR Integration — Frontend Flow

When admin clicks "Run Proof of Reasoning":

1. Frontend calls existing /step/resolve via proxy
   (same as playground single-call mode)
   Request: {
     question: market.question,
     providers: [user-selected or default],
     collectors: [user-selected or default]
   }

2. Show loading spinner with "Running PoR pipeline..."
   (reuse playground pipeline-progress component)

3. On success, display RunSummary:
   - Outcome + confidence (prominently)
   - Evidence bundles (collapsible, reuse detail components)
   - Reasoning steps (collapsible)
   - PoR root hash

4. Pre-fill resolution form:
   - Outcome → from RunSummary
   - Confidence → from RunSummary
   - por_root → from RunSummary
   - method → "por" (changes to "por_modified" if admin edits)

5. Admin can:
   - Accept as-is → method stays "por"
   - Modify outcome/confidence → method becomes "por_modified"
   - Add admin_notes explaining override

Note: No new API needed for PoR — reuses existing oracle endpoints.
The ONLY new endpoint is POST /admin/markets/:id/resolve for
the final confirmed submission.
